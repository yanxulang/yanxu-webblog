# 从零理解言枢博客

## 1. 数据与页面上下文

`数据.yx`返回文章典；`页面.yx`把它们整理为言标需要的上下文，并使用命名路由生成地址：

```yanxu
置 文章【「url」】为 上下文.反向（「文章详情」，{「id」：文章【「id」】}）；
```

模板因此只读取`文章.url`，无需知道路由模式。

## 2. 基础模板与继承

`base.yb.html`包含共享 head、header 和 footer，并定义`主体`块。`home.yb.html`用`承`和`块`覆盖页面主体；文章列表使用`逐`，所有动态文字使用自动转义插值。

文章正文包含`<标签>`，详情页中的`{{ 文章.body }}`会输出`&lt;标签&gt;`。这是言标默认安全边界，不需要每次手工调用转义法。

## 3. 配置化应用

`应用.yx`在工厂内创建配置：

```yanxu
定 配置项 为 言枢.配置（）
    .设应用名（「言枢小笺」）
    .设静态根（静态根）
    .设模板根（模板根）；
定 所应用 为 言枢.创建应用（配置项）；
```

框架据此建立模板环境和默认安全首部中间件。

## 4. 命名路由组

页面路由使用独立名称；API 使用`/api`前缀和`api`命名空间：

```yanxu
所应用.命名取（「/posts/:id」，「文章详情」，文章处理）；
定 接口 为 所应用.命名路由组（「/api」，「api」）；
接口.命名取（「/posts」，「文章列表」，JSON接口）；
```

对应反向名称为`文章详情`和`api:文章列表`。

## 5. 页面与 API

页面处理器调用`应用.模板响应`；文章不存在时调用同一个自定义 404 处理器。JSON 由`言枢.JSON响应`序列化，言据先由`言据.序列化`生成正文，再交给言枢设置媒体类型。

静态文件由`挂配置静态`使用配置中的静态根，路径防护拒绝绝对路径、反斜杠、NUL、`.`和`..`。

## 6. 无端口集成测试

```yanxu
定 客户端 为 言枢.测试客户端（所应用）；
定 首页 为 客户端.取（「/」）；
定 不许 为 客户端.请求（「POST」，「/api/posts」）；
```

规格覆盖首页、模板自动转义、自定义 404、JSON、言据、静态 CSS、目录穿越、405/Allow 和反向 URL。

## 7. 练习

1. 新增第三篇文章，观察首页、详情、JSON 和言据同步变化；
2. 新建`about.yb.html`并注册命名路由；
3. 新增`POST /api/posts`，用`上下文.JSON正文（）`读取输入；
4. 为文章列表加入查询过滤，并测试`查询全部`；
5. 把`数据.yx`替换为受校验的文件存储。

生产限制见[运行与部署](running-and-deployment.md)。
