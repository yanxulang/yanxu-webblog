# 言枢博客项目结构

项目使用“领域数据 → 页面数据 → 言标模板 → 言枢应用 → 进程入口”的单向依赖。

| 路径 | 职责 |
| --- | --- |
| `src/数据.yx` | 返回与查询文章领域数据。 |
| `src/页面.yx` | 建立模板上下文，并通过命名路由生成 URL。 |
| `templates/base.yb.html` | 基础文档结构与`主体`块。 |
| `templates/home/post/404.yb.html` | 继承基础模板的页面。 |
| `templates/partials/` | 可包含的 head、header 和 footer。 |
| `src/应用.yx` | 配置、处理器、API、路由组、错误与静态路由。 |
| `src/主.yx` | 监听地址、项目路径和服务器生命周期。 |
| `tests/集成.yx` | 无端口端到端应用测试。 |

页面数据层不生成 HTML，模板不读取存储。这样可以独立替换数据源、模板外观或 API，而不让边界互相渗透。

## 依赖

应用源码只直接引入它实际使用的`yanxu-web`与`言据`。`yanxu-html`和`yanxu-http`由言枢作为传递依赖管理；业务项目不再为模板节点工厂直接依赖 HTML 包。

`言序.lock`由言包生成并固定言枢 0.2 的版本、精确 Git 修订和内容校验。升级时先发布言枢、再执行`yanbao update`更新博客，不能手工填写修订与校验和。

## 扩展位置

- 新页面：新增`.yb.html`并在`页面.yx`建立上下文；
- 新 API：放入独立处理器，数量增长后拆出`src/接口.yx`；
- 新业务分区：用`命名路由组`统一前缀与命名空间；
- 持久化：替换`数据.yx`，不要让模板直接读取文件或数据库；
- 认证：用中间件建立身份上下文，并在处理器做授权；
- 表单：解析与验证应在独立领域模块完成，模板只显示已整理值和错误。

## 测试

首选`言枢.测试客户端`驱动应用。只有套接字、HTTP 字节或部署集成需要验证时，才用`--once`保留少量真实连接冒烟测试。
